# CLI Agent 测试用例

> 基于当前项目实际实现（DeepSeek V4 Flash · 3 工具 · SessionManager · ContextManager · logger_config）

**结果标记说明：**

| 标记  | 含义          |
|:---:|:----------- |
| ✅   | 通过          |
| ❌   | 失败（失败原因写备注） |
| ⏳   | 待测试         |
| ➖   | 不适用/暂不测     |

---

## 模块 1：基本循环

| 编号          | 标题        | 步骤                   | 预期                                                          | 结果  |
|:----------- |:--------- |:-------------------- |:----------------------------------------------------------- |:---:|
| TC-LOOP-001 | 纯对话（无工具）  | 输入 `你好`              | LLM 流式输出思考+正文，无工具调用，回到用户输入                                  | ✅   |
| TC-LOOP-002 | 单工具调用     | 输入 `计算 25*4`         | 调用 `calculator_tool(expression="25*4")`，返回结果 `100`，LLM 汇总回复 | ✅   |
| TC-LOOP-004 | 工具执行中用户拒绝 | 触发 calculator，输入 `R` | 工具结果 = "工具调用被用户取消执行"，LLM 据此继续                               | ✅   |

## 模块 2：工具注册与调用

| 编号          | 标题                | 步骤                          | 预期                                         | 结果  |
|:----------- |:----------------- |:--------------------------- |:------------------------------------------ |:---:|
| TC-TOOL-001 | CLI 工具正常执行        | 输入 `执行 dir 命令`              | 调用 `run_cli_tool(command="dir")`，返回 stdout | ✅   |
| TC-TOOL-002 | CLI 工具超时          | 输入 `调用CLI工具并设置极小的timeout测试` | 返回 `timed_out=True`，LLM 提示超时               | ✅   |
| TC-TOOL-003 | calculator 正常     | 输入 `sqrt(16)+1`             | 返回 `5.0`                                   | ✅   |
| TC-TOOL-004 | calculator 除零     | 输入 `1/0`                    | 返回 error="除零错误"，LLM 友好提示                   | ✅   |
| TC-TOOL-005 | calculator 不安全表达式 | 输入含 `__import__` 的表达式       | 返回 error，计算被拦截                             | ✅   |
| TC-TOOL-006 | search mock 命中    | 输入 `搜索 python`              | 返回预设的 Python 相关结果                          | ✅   |
| TC-TOOL-007 | search mock 未命中   | 输入随机词                       | 返回动态填充的默认结果                                | ✅   |

## 模块 3：LLM 流式解析

|:------------ |:----------- |:------------- |:--------------------------------------------------------- |:---:|

## 模块 4：会话管理（SessionManager）

| 编号          | 标题              | 步骤                 | 预期                                           | 结果  |
|:----------- |:--------------- |:------------------ |:-------------------------------------------- |:---:|
| TC-SESS-001 | 新建会话            | 菜单选 `1`            | 生成 `sessions/YYYYMMDD_HHMMSS.json`，仅含 system | ✅   |
| TC-SESS-002 | `/save` 保存      | 对话中输 `/save`       | 保存当前 messages 到文件，打印确认                       | ✅   |
| TC-SESS-003 | `/exit` 保存并返回菜单 | 输入 `/exit`         | 保存后回到菜单首页                                    | ✅   |
| TC-SESS-004 | Ctrl+C 自动保存     | 对话中按 Ctrl+C        | 保存后返回菜单，不崩溃                                  | ✅   |
| TC-SESS-005 | 继续已有会话          | 菜单选 `2` + 选择编号     | 加载文件，system prompt 被程序常量覆盖，消息数正确             | ✅   |
| TC-SESS-006 | 管理删除会话          | 菜单选 `3` + 编号 + `y` | 文件从 `sessions/` 删除                           | ✅   |
| TC-SESS-007 | 恢复损坏文件          | 手动损坏 JSON          | 损坏的文件不显示                                     | ✅   |
| TC-SESS-008 | 未知指令            | 输入 `/xxx`          | 提示 `[未知指令]`，不中断                              | ✅   |

## 模块 7：异常与边界

| 编号         | 标题          | 步骤               | 预期                                        | 结果  |
|:---------- |:----------- |:---------------- |:----------------------------------------- |:---:|
| TC-ERR-001 | LLM API 不可用 | 断开网络后发送消息        | `stream_query` 抛异常，捕获后打印 `[LLM 调用异常]`，不崩溃 | ✅   |
| TC-ERR-002 | 空输入         | 输入空行（直接回车）       | 等待重新输入，不提交给 LLM                           | ✅   |
| TC-ERR-003 | 菜单无效输入      | 输入 `5` / `abc`   | 提示 `[无效选择] 请输入 1-4`                       | ✅   |
| TC-ERR-004 | 工具参数解析失败    | 模拟 LLM 返回非法 JSON | `json.loads` 抛异常，`tool_execution` 中捕获，不中断 | ⏳   |
