# CLI Agent — 基于 DeepSeek 的 ReAct 智能助手

## 运行方式

```bash
python Agent.py
```

程序启动后会显示菜单：

```
  1. 新建对话
  2. 继续已有对话
  3. 管理会话（查看/删除）
  4. 退出
```

## 系统设计

### 架构概览

```
┌─────────────────────────────────────────────────────────┐
│                     Agent.py (main)                      │
│  ReAct 循环：用户输入 → LLM 调用 → 工具执行 → 结果反馈    │
├───────────────────┬─────────────────────┬───────────────┤
│  SessionManager   │   ContextManager    │   logger_config│
│  (会话持久化)      │   (轮次压缩)         │   (日志记录)   │
├───────────────────┴─────────────────────┴───────────────┤
│                     Tools/                                │
│  ┌─────────────┐  ┌───────────────┐  ┌───────────────┐  │
│  │  CLI_Tool   │  │ Calculator_Tool│  │  Search_Tool  │  │
│  │  (命令行执行)│  │  (安全计算)    │  │  (Mock 搜索)  │  │
│  └─────────────┘  └───────────────┘  └───────────────┘  │
├─────────────────────────────────────────────────────────┤
│                    config.py                              │
│              deepseek_api_key = os.getenv(...)            │
└─────────────────────────────────────────────────────────┘
```

### ReAct 循环流程

```
                      ┌──────────────────────┐
                      │   启动菜单            │
                      │  (新建/继续/管理/退出) │
                      └──────────┬───────────┘
                                 │
                      ┌──────────▼───────────┐
                      │   会话初始化          │
                      │  (加载/空消息列表)     │
                      └──────────┬───────────┘
                                 │
                      ┌──────────▼───────────┐
          ┌───────────┤   压缩检查            │
          │           │  (needs_compression)  │
          │           └──────────┬───────────┘
          │                      │
          │           ┌──────────▼───────────┐
          │           │   获取用户输入        │<————————|
          │           │  (/save, /exit, ...) │        |
          │           └──────────┬───────────┘        |
          │           ┌──────────▼───────────┐        ↑
          │           │   输入是否为命令？     │————>是，执行命令
          │           │  (是否以“/”作为开头)   │
          │           └──────────┬───────────┘
          │                      │否，普通消息输入
          │           ┌──────────▼───────────┐
          │           │   LLM 流式调用        │
          │           │  (stream_query)      │
          │           └──────────┬───────────┘
          │                      │
          │           ┌──────────▼───────────┐
          │           │   解析流式输出         │
          │           │  (正文/思考/工具调用)  │
          │           └──────────┬───────────┘
          │                      │
          │           ┌──────────▼───────────┐
          │    有工具  │   是否有工具调用?     │──── 无工具 ──→ 输出回复，回到顶部
          │           └──────────┬───────────┘
          │                      │ 有工具
          │           ┌──────────▼───────────┐
          └───────────┤   执行工具 (用户确认) │
                      │  (tool_execution)    │
                      └─────────────────────┘
```

### 工具注册机制

通过 `get_tools_info()` 返回工具的 name、description、parameters schema（Function Calling 格式），
通过 `Function_Name_Mapping` 字典映射到实际执行函数。

| 工具名称              | 映射函数        | 功能      |
|:----------------- |:----------- |:------- |
| `run_cli_tool`    | `run_cli`   | 执行命令行命令 |
| `calculator_tool` | `calculate` | 安全数学计算  |
| `search_tool`     | `search`    | Mock 搜索 |

用户在执行工具前可干预：

- **回车**：正常执行
- **R**：拒绝执行，返回"工具调用被用户取消执行"
- **A**：执行并追加额外信息
- **RA**：拒绝并追加信息

## 会话管理 (SessionManager) — 存储与召回

### 存储位置

会话以 JSON 文件存储于项目根目录 `sessions/` 下，文件名 = 时间戳 `YYYYMMDD_HHMMSS.json`。

### 文件结构

```json
{
  "session_id": "20250628_101500",
  "session_name": "你好，我想了解...",
  "messages": [
    {"role": "system", "content": "..."},
    {"role": "user", "content": "你好"},
    {"role": "assistant", "content": "你好！"}
  ]
}
```

### 存储时机（写入文件）

| 触发方式       | 行为                |
|:---------- |:----------------- |
| `/save` 指令 | 立即保存当前完整 messages |
| `/exit` 指令 | 保存后返回启动菜单         |
| Ctrl+C     | 自动保存后返回启动菜单       |

每次保存都是**全量覆写** JSON 文件（`open("w")`），不是增量追加。

### 召回时机（读取文件）

| 场景          | 行为                  |
|:----------- |:------------------- |
| 菜单选「继续已有对话」 | 从文件加载完整 messages 列表 |
| 文件损坏        | 提示 `[文件已损坏]`，返回菜单   |

### 召回时的关键逻辑

加载旧会话时，**程序硬编码的 `SYSTEM_PROMPT` 常量会覆盖文件中保存的 system prompt**，确保即使后续修改了 system prompt，旧会话也能使用最新版本。

## 上下文压缩 (ContextManager) — 轮次管理

### 触发时机

**每次用户输入前**，在 while 循环顶部检查。

### 判断条件

跳过 system prompt，统计 `role == "user"` 的消息数量 > `max_rounds`（默认 10）。

### 压缩流程

```
need_compression? → 是 → 追加摘要模板user消息 → 非流式调LLM
                             │ 成功           │ 失败（返回None）
                             ▼                ▼
                        compress()       跳过压缩，原messages不变（可以有优化空间）
                             │
                             ▼
               [system, 摘要user, ...最新一轮(工具截断)]
```

### 最新一轮处理

- 找到最后一条 user 消息的位置，保留该轮完整对话
- 工具返回结果（`role="tool"` 的 `content`）超过 30 字时截断为 `前30字...`

### 失败保护

LLM 摘要生成失败（网络异常、API 超时等）时，`_get_summary` 返回 `None`，程序**跳过本轮压缩**，原 messages 不变，流程不中断。

## 日志系统 (logger_config)

### 文件位置

`logs/YYYYMMDD_HHMMSS.log`，每次程序启动生成一个独立文件。

### 日志格式

```
2025-06-28 10:15:30 - [INFO] - [Agent] - 程序启动
2025-06-28 10:15:35 - [INFO] - [SessionManager] - 会话已保存, session_id=...
2025-06-28 10:15:40 - [DEBUG] - [Tools.CLI_Tool] - CLI 执行开始: command='dir'
2025-06-28 10:15:45 - [WARNING] - [Agent] - 未知指令: /xxx
2025-06-28 10:15:50 - [ERROR] - [Agent] - LLM 请求失败: ConnectionError
```

### 事件等级

| 等级      | 用途                            |
|:-------:|:----------------------------- |
| DEBUG   | 工具内部执行细节（开始/完成/耗时）            |
| INFO    | 关键业务流程（程序启停、会话操作、LLM 调用、压缩完成） |
| WARNING | 非致命异常（未知指令、工具被拒、压缩跳过）         |
| ERROR   | 功能级故障（LLM 不可用、文件损坏、工具执行异常）    |
